Introduction

The time complexity of an algorithm is the amount of time taken by such algorithm to run as a function of its input. This is, for an algorithm with a complexity of 2n it takes with an input of size n, 2n units of time to complete its execution, this is often denoted as O(2n). Using an expressive enough type system is possible to implement functions with associated complexities embedded in its type. One example of this is Danielsson [2008] which implements a library (THUNK) to add complexity annotations to functions in Agda.

The aim of this programming lab is to implement various functions using THUNK to annotate its complexity.

Assignment (80%)

For a description of THUNK, see Danielsson [Section 3, 2008].
For an example of how to use THUNK, see Complexity.agda.

  • Define a function sum : (a : ℕ) → (b : ℕ) → ℕ-thunk (1 + a) that adds two natural numbers.
  • Define a function max : (n : ℕ) → (m : ℕ) → ℕ-thunk (1 + min' n m) that returns the greatest of two natural numbers.
  • Define a function min : (n : ℕ) → (m : ℕ) → ℕ-thunk (1 + min' n m) that returns the smallest of tow natural numbers.
  • Define a function length : {A : Set} → (l : List A) → ℕ-thunk (1 + length' l) that given a list returns its length.
  • (10%) Annotate your Agda source code.

Please note that the following conventions must be followed for the implementation of any function using THUNK to be correct:

  • Every run-time function clause (including those of anonymous lambdas) has to start with ✓ .
  • The function force may not be used in run-time terms.
  • Library functions may not be used partially applied.

Please follow this conventions, otherwise your implementation will not be considered correct.
